About

Groupie Tracker - Harry Potter Edition

Project Presentation

Welcome to the Harry Potter Groupie Tracker, a web application that brings the magical universe of Harry Potter to life through an interactive and user-friendly interface. This project was developed as part of the Groupie Tracker assignment at Ynov Campus, demonstrating mastery of Go web development, REST API consumption, and clean architecture design.

The application allows users to explore over 600 characters, browse the complete collection of books and movies, discover hundreds of spells and potions, and manage their favorite characters. Built entirely with Go's standard library and vanilla HTML/CSS, this project showcases how powerful native tools can be when properly utilized.

Technical Overview

Web app built with Go using Potter DB API. Features: character search, filters, favorites, pagination, catalogs for books/movies/spells/potions.

Tech stack: Go (standard library only), HTML, CSS. No frameworks, no external dependencies.

Installation

git clone <repo-url>
cd API_HP/src/cmd
go run main.go

Access: http://localhost:8080

Main Routes

  • / - Home
  • /search - Search with filters
  • /character/{id} - Character details
  • /favorites - Favorites list
  • /books, /movies, /spells, /potions - Catalogs

API

Potter DB v1: https://api.potterdb.com/v1

Endpoints used: /characters, /books, /movies, /spells, /potions

FAQ

How was the project decomposed?

6 phases: Analysis & Design → Backend → Core Features → Advanced Features → Error Handling → Design & Docs

Task distribution?

Solo project: 40% backend, 35% frontend, 15% debugging, 10% documentation

Time management?

~80-100 hours over 4 weeks. MVP first, then bonus features.

Documentation strategy?

Official Go docs, Potter DB documentation, MDN Web Docs, CSS-Tricks
